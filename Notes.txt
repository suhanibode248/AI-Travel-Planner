1. PROJECT DESCRIPTION
================================================================================

AI Travel Planner is a web-based application that revolutionizes travel planning
by leveraging Artificial Intelligence to generate personalized travel itineraries.
Instead of spending hours researching destinations, hotels, restaurants, and 
activities across multiple websites, users can get a complete customized travel 
plan within seconds.

The application accepts user inputs such as destination, budget, trip duration,
and travel preferences, then uses the Anthropic Claude AI model to generate a
comprehensive, personalized itinerary. The system provides daily schedules,
activity recommendations, dining suggestions, transportation tips, and estimated
expenses.

Key Benefits:
  • Saves time - Get complete travel plans in seconds
  • Personalized - Customized based on budget, interests, and preferences
  • User-friendly - Simple interface requiring minimal effort
  • Shareable - Users can share generated itineraries via unique links
  • Persistent - Registered users can save and access their trips anytime

Project Name: VOYAGER AI (Voyager - because it plans voyages/travels)
University: Rashtrasant Tukadoji Maharaj Nagpur University, Nagpur
Academic Focus: AI in Tourism and Travel Planning Services

3. HOW TO RUN PROJECT
================================================================================

INSTALLATION STEPS:

1. Clone or Extract Project:
   $ unzip ai-travel-planner.zip
   $ cd ai-travel-planner

2. Create Virtual Environment (Recommended):
   $ python3 -m venv venv
   $ source venv/bin/activate  (On Windows: venv\Scripts\activate)

3. Install Dependencies:
   $ pip install -r requirements.txt

4. Configure Environment Variables:
   Create a .env file in project root:
   
   ANTHROPIC_API_KEY=your_api_key_here
   SECRET_KEY=your_secret_key_here
   MAIL_SERVER=smtp.gmail.com
   MAIL_PORT=587
   MAIL_USE_TLS=True
   MAIL_USERNAME=your_email@gmail.com
   MAIL_PASSWORD=your_app_password
   MAIL_DEFAULT_SENDER=noreply@voyager.ai
   FLASK_ENV=production
   DEBUG=False

5. Initialize Database:
   $ python3 -c "from app import app, db; app.app_context().push(); db.create_all()"

6. Run Application:
   $ python3 app.py
   or
   $ flask run

7. Access Application:
   Open browser and go to: http://localhost:5000

8. Default Admin Credentials:
   Username: admin
   Password: admin123
   (Change password after first login for security)

DEPLOYMENT OPTIONS:

Option A - Vercel (Recommended for ease):
  • Push code to GitHub
  • Connect repository to Vercel
  • Add environment variables in Vercel dashboard
  • Deploy automatically

Option B - Heroku:
  • Create Procfile with: web: python app.py
  • Install Heroku CLI
  • Deploy with: git push heroku main

Option C - Local Server:
  • Run with Gunicorn: gunicorn -w 4 app:app
  • Use Nginx as reverse proxy
  • Configure SSL/TLS certificates

TROUBLESHOOTING:

If port 5000 is in use:
  $ python3 app.py --port 8000

If database won't initialize:
  $ rm -f instance/voyager.db
  $ python3 -c "from app import app, db; app.app_context().push(); db.create_all()"

If imports fail:
  $ pip install --upgrade pip
  $ pip install -r requirements.txt --force-reinstall

4. HOW IT WORKS (TECH FLOW)
================================================================================

USER JOURNEY:

1. User Visits Application:
   → Lands on index.html (homepage)
   → Can choose to "Generate Trip" (guest) or "Login" (registered user)

2. Trip Creation Process:
   User Input Form:
   ├─ Destination (text input + autocomplete from cities.json)
   ├─ Duration (days)
   ├─ Budget (currency amount)
   ├─ Trip Type (adventure, relaxation, cultural, family, etc.)
   ├─ Interests (multiple selection)
   └─ Special Requirements (dietary, mobility, etc.)

3. Frontend Processing (script.js):
   → Validates user inputs
   → Generates unique session ID for guest users
   → Sends data to /api/generate-plan endpoint via FETCH

4. Backend Processing (app.py):
   → Receives request at /api/generate-plan route
   → Constructs detailed AI prompt with user inputs
   → Calls Anthropic Claude API with prompt
   → Receives AI-generated itinerary (structured JSON format)

5. AI Model (Claude via Anthropic):
   → Receives detailed prompt with travel requirements
   → Generates personalized itinerary including:
      • Daily schedule and activities
      • Restaurant recommendations from restaurants.json
      • Hotel suggestions from hotels.json
      • Transportation options from transport.json
      • Budget breakdown
      • Travel tips and warnings
   → Returns response in structured format

6. Response Processing:
   Backend:
   → Parses AI response
   → Validates and formats itinerary data
   → Saves trip to database (if user is registered)
   → Returns JSON response to frontend

   Frontend:
   → Receives and displays itinerary
   → Renders with dynamic HTML/CSS
   → Enables sharing/saving/exporting options

7. Database Storage (SQLAlchemy + SQLite):
   User Model:
   ├─ id (Primary Key)
   ├─ username (unique)
   ├─ email (unique)
   ├─ password (bcrypt hashed)
   ├─ name
   └─ created_at

   SavedTrip Model:
   ├─ id (Primary Key)
   ├─ user_id (Foreign Key to User)
   ├─ share_id (unique share link identifier)
   ├─ destination
   ├─ days
   ├─ budget
   ├─ trip_type
   ├─ plan_json (entire itinerary as JSON)
   └─ created_at

  • Python 3.x

  • Flask extensions (SQLAlchemy, Bcrypt, Mail, Limiter)
  • Anthropic Python SDK

Database:
  • SQLite for data persistence
  • SQLAlchemy ORM for database abstraction

External APIs:

Data Flow Summary:
User Input → Validation → AI Prompt Construction → Claude API Call →
Response Parsing → Database Save → Frontend Display

6. COMMON ISSUES & FIXES
================================================================================

ISSUE #1: "ModuleNotFoundError: No module named 'flask'"
SOLUTION:
  • Ensure virtual environment is activated
  • Run: pip install -r requirements.txt
  • Check Python version: python3 --version (must be 3.8+)
  • Try: pip install --upgrade pip

ISSUE #2: "API Key Invalid" error
SOLUTION:
  • Verify ANTHROPIC_API_KEY in .env file is correct
  • Ensure API key has appropriate permissions
  • Check key hasn't expired or been revoked
  • Regenerate key from Anthropic dashboard if needed
  • Ensure no extra spaces or quotes around key

ISSUE #3: Database locked error (OperationalError)
SOLUTION:
  • Close any other instances of the application
  • Delete instance/voyager.db and restart
  • Use: SQLiteDatabase.close() to properly close connections
  • Restart Flask development server

ISSUE #4: Port 5000 already in use
SOLUTION:
  • Use different port: python3 app.py --port 8000
  • Or kill process: lsof -ti:5000 | xargs kill -9
  • Check if another Flask app is running

ISSUE #5: Email sending not working
SOLUTION:
  • Verify MAIL_SERVER and credentials in .env
  • Enable "Less secure apps" if using Gmail
  • Use Gmail App Passwords instead of main password
  • Check firewall settings allow SMTP (port 587)
  • Verify MAIL_USERNAME and MAIL_PASSWORD are correct

ISSUE #6: Static files (CSS/JS) not loading
SOLUTION:
  • Clear browser cache (Ctrl+Shift+Del)
  • Ensure static/ folder exists in project root
  • Check Flask static folder configuration
  • Run: python3 app.py with DEBUG=True

ISSUE #7: "CORS error" when calling APIs
SOLUTION:
  • Add CORS headers to Flask app (if needed)
  • Ensure API calls are from same origin
  • Check firewall allowing API requests
  • Verify API endpoints are correct in script.js

ISSUE #8: Login/Registration not working
SOLUTION:
  • Clear browser cookies
  • Verify email is unique in database
  • Check password meets requirements
  • Reset database: rm -f instance/voyager.db
  • Verify bcrypt is installed correctly

ISSUE #9: AI response timeout
SOLUTION:
  • Check internet connection
  • Verify Anthropic API is accessible
  • Reduce prompt complexity
  • Increase request timeout in app.py
  • Check API rate limits haven't been exceeded

ISSUE #10: Shared trip link returns 404
SOLUTION:
  • Verify share_id is valid and exists in database
  • Check share_id wasn't deleted from database
  • Regenerate share link from dashboard
  • Ensure database file (voyager.db) is not corrupted

ISSUE #11: Strange characters or encoding errors
SOLUTION:
  • Ensure file encoding is UTF-8
  • Check database UTF-8 configuration
  • Update Python: python3 -m pip install --upgrade python
  • Recreate database files

ISSUE #12: Application running very slow
SOLUTION:
  • Check server resources (CPU, RAM, Disk)
  • Optimize AI prompt length
  • Enable database indexing
  • Use caching for static files
  • Consider upgrading to faster server

GENERATING BETTER ITINERARIES:

1. Be Specific with Preferences:
   ✓ Instead of "Adventure trip" say "Rock climbing and hiking"
   ✓ Mention specific interests (photography, food, history, etc.)
   ✓ Include dietary restrictions or allergies
   ✓ Specify mobility limitations if any

2. Use Realistic Budgets:
   ✓ Research average costs for your destination
   ✓ Include flights, hotels, meals, activities
   ✓ Add buffer for unexpected expenses (10-15%)
   ✓ Consider currency exchange rates

3. Optimal Trip Duration:
   ✓ Minimum 2 days for meaningful itinerary
   ✓ Ideal: 5-7 days for complete experience
   ✓ Maximum: 30 days (algorithm handles longer trips)
   ✓ Include travel days in duration count

4. Choose Correct Trip Type:
   ✓ Adventure - Hiking, sports, outdoor activities
   ✓ Relaxation - Beaches, spas, resort experiences
   ✓ Cultural - Museums, historical sites, local experiences
   ✓ Family - Kid-friendly activities, safe locations
   ✓ Business - Conference locations, business amenities
   ✓ Luxury - High-end restaurants, premium hotels

5. Refine Your Preferences:
   ✓ Select all relevant interests for better recommendations
   ✓ Mention traveling with family, friends, or solo
   ✓ Include any religious or cultural preferences
   ✓ Specify if you're a first-time visitor

6. Review and Customize Results:
   ✓ Check AI suggestions before finalizing
   ✓ Adjust daily schedules to your pace
   ✓ Verify recommendations match your interests
   ✓ Feel free to combine with other travel resources
